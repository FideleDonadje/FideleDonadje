## 👋 About Me

I'm a cloud security engineer focused on AWS — building tools that turn raw security findings into actionable intelligence. My work sits at the intersection of **cloud security**, **AI automation**, and **compliance engineering**.

- 🔭 Currently building an **ATO (Authority to Operate) Assist Mode** — auto-generates NIST 800-53 control statements and POA&M entries from Security Hub findings using AWS Bedrock
- ⚡ Everything I build is infrastructure-as-code first (AWS CDK), production-grade, and security-reviewed
- 🧠 Learning-first mindset: I design before I build, and I own every architectural decision

## 🚀 Featured Projects

### 🔐 [security-triage-agent](https://github.com/FideleDonadje/security-triage-agent)
> AI-powered AWS Security Operations Center triage agent

A production-grade system that ingests security findings and routes them through an intelligent triage workflow. Built with **AWS Bedrock AgentCore** for agent runtime, backed by **DynamoDB Streams** for event-driven task queuing, and deployed entirely via **CDK**.

`AWS Bedrock` `AgentCore` `CDK` `Lambda` `DynamoDB` `Cognito` `React/Vite` `TypeScript`

---

### 🛡️ [security-control-coverage-analyzer](https://github.com/FideleDonadje/security-control-coverage-analyzer)
> Account-level AWS security control coverage auditing

Produces auditable JSON/Markdown reports showing exactly which AWS security controls are active vs. missing — no dashboards, no remediation noise. Designed for compliance engineers who need clean evidence artifacts.

`Python` `AWS Security Hub` `NIST 800-53` `Markdown Reports`
